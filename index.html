# 概览

因果API 是一个统一的 AI 模型接口平台，提供对 OpenAI、Anthropic、Google、DeepSeek 等主流 AI 模型的统一访问。 所有接口均兼容 OpenAI API 格式，只需修改 base_url 即可无缝切换。

💡 核心优势
  • 一个 API Key 调用所有模型
  • 完全兼容 OpenAI SDK
  • 支持流式输出（SSE）
  • 更低的价格，更高的稳定性

# 鉴权方式

所有 API 请求需要在 HTTP Header 中携带 API Key 进行身份验证。

🔑 请求头格式
复制 http 
Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
📋 获取 API Key
  1. 访问 控制台 并登录
  2. 进入「令牌管理」页面
  3. 点击「创建新令牌」
  4. 复制生成的 API Key（以 sk- 开头）
⚠️ 安全提示 请妥善保管你的 API Key，不要将其泄露到公开代码仓库或客户端代码中。

# 聊天补全

支持多轮对话的聊天接口，兼容 OpenAI Chat Completions 格式。支持 GPT-4o、Claude、Gemini、DeepSeek 等主流模型。

POST /v1/chat/completions 热门

# 请求参数

参数类型必填说明
modelstring是模型 ID，如 gpt-4o、claude-sonnet-4-6
messagesarray是消息数组，包含 role 和 content
temperaturenumber否采样温度，0-2，默认 1
max_tokensinteger否最大生成 token 数
streamboolean否是否流式输出，默认 false
top_pnumber否核采样参数，默认 1
frequency_penaltynumber否频率惩罚，-2 到 2
presence_penaltynumber否存在惩罚，-2 到 2
toolsarray否可用工具列表（Function Calling）
tool_choicestring/object否工具选择策略

# 消息格式

角色说明
system系统提示词，设定 AI 行为
user用户输入
assistantAI 回复
tool工具调用结果

# 示例请求

复制 python 
from openai import OpenAI

client = OpenAI(
    api_key="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    base_url="https://api.causmatrix.xyz/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一个有用的助手"},
        {"role": "user", "content": "你好，请介绍一下自己"}
    ],
    temperature=0.7,
    max_tokens=1000
)

print(response.choices[0].message.content)
复制 bash 
curl https://api.causmatrix.xyz/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "你是一个有用的助手"},
      {"role": "user", "content": "你好，请介绍一下自己"}
    ],
    "temperature": 0.7
  }'

# 流式输出

设置 stream: true 启用 SSE 流式输出：

复制 python 
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "写一首诗"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")

# 多模态输入

支持图片和文本混合输入（适用于 GPT-4o、Gemini 等多模态模型）：

复制 python 
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "这张图片里有什么？"},
                {
                    "type": "image_url",
                    "image_url": {"url": "https://example.com/image.jpg"}
                }
            ]
        }
    ]
)

# Function Calling

支持工具调用，让模型能够调用外部函数：

复制 python 
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的天气信息",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {
                        "type": "string",
                        "description": "城市名称"
                    }
                },
                "required": ["city"]
            }
        }
    }
]

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "北京今天天气怎么样？"}],
    tools=tools,
    tool_choice="auto"
)

# 文本嵌入

将文本转换为向量表示，用于语义搜索、文本相似度计算、聚类分析等场景。

POST /v1/embeddings 热门

# 请求参数

参数类型必填说明
modelstring是嵌入模型 ID
inputstring/array是要嵌入的文本或文本数组
encoding_formatstring否返回格式：float 或 base64

# 示例请求

复制 python 
response = client.embeddings.create(
    model="text-embedding-3-small",
    input="因果API 是一个统一的 AI 模型接口平台"
)

# 获取向量
embedding = response.data[0].embedding
print(f"向量维度: {len(embedding)}")
print(f"前5个值: {embedding[:5]}")
复制 python 
# 批量嵌入
response = client.embeddings.create(
    model="text-embedding-3-small",
    input=[
        "第一段文本",
        "第二段文本",
        "第三段文本"
    ]
)

# 获取所有向量
for item in response.data:
    print(f"Index {item.index}: {len(item.embedding)} 维")

# 语义搜索示例

复制 python 
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 获取查询和文档的向量
query_embedding = client.embeddings.create(
    model="text-embedding-3-small",
    input="如何使用 API？"
).data[0].embedding

doc_embeddings = client.embeddings.create(
    model="text-embedding-3-small",
    input=["API 使用指南", "快速入门教程", "常见问题解答"]
).data

# 计算相似度
for doc in doc_embeddings:
    similarity = cosine_similarity(query_embedding, doc.embedding)
    print(f"文档 {doc.index}: 相似度 {similarity:.4f}")

# 音频处理

支持语音识别（ASR）、音频翻译等功能。支持 Whisper、Gemini 等音频模型。

POST /v1/audio/transcriptions 热门

将音频转换为文本（语音识别）。

# 请求参数

参数类型必填说明
filefile是音频文件（支持 mp3、mp4、mpeg、mpga、m4a、wav、webm）
modelstring是模型 ID，如 whisper-1
languagestring否音频语言（ISO-639-1 格式）
promptstring否提示词，用于引导模型的风格
response_formatstring否返回格式：json、text、srt、verbose_json、vtt
temperaturenumber否采样温度，0-1

# 示例请求

复制 python 
with open("audio.mp3", "rb") as audio_file:
    transcript = client.audio.transcriptions.create(
        model="whisper-1",
        file=audio_file,
        language="zh",
        response_format="text"
    )

print(transcript.text)
复制 bash 
curl https://api.causmatrix.xyz/v1/audio/transcriptions \
  -H "Authorization: Bearer sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" \
  -F file="@audio.mp3" \
  -F model="whisper-1" \
  -F language="zh" \
  -F response_format="text"
POST /v1/audio/translations

将音频翻译为英文。

# 音频翻译示例

复制 python 
with open("chinese_audio.mp3", "rb") as audio_file:
    translation = client.audio.translations.create(

# 内容审查

对文本内容进行安全审核，检测是否包含暴力、仇恨、自残、色情等违规内容。

POST /v1/moderations

分析文本内容是否违反使用政策。

# 请求参数

参数类型必填说明
modelstring是审查模型 ID，如 text-moderation-latest
inputstring/array是要审查的文本

# 检测类别

类别说明
hate仇恨言论
hate/threatening仇恨威胁
harassment骚扰
harassment/threatening骚扰威胁
self-harm自残
self-harm/intent自残意图
self-harm/instructions自残指导
sexual色情内容
sexual/minors涉及未成年人的色情内容
violence暴力内容
violence/graphic血腥暴力

# 示例请求

复制 python 
response = client.moderations.create(
    model="text-moderation-latest",
    input="这段文本需要进行内容审查"
)

result = response.results[0]

# 检查是否被标记
if result.flagged:
    print("内容被标记为违规")
    # 查看具体类别

        if details:
            print(f"- {category}: {details}")
else:
    print("内容通过审查")

# 错误码

API 返回标准 HTTP 状态码，错误响应体包含详细的错误信息。

状态码说明常见原因
200成功请求成功处理
400请求错误参数缺失或格式错误
401未授权API Key 无效或缺失
403禁止访问没有权限访问该资源
404未找到请求的资源不存在
429请求过多触发速率限制
500服务器错误服务器内部错误
502网关错误上游服务不可用
503服务不可用服务器过载或维护

# 速率限制

为保证服务稳定性，API 对请求频率进行限制。

限制维度默认值说明
每令牌 RPM60 次/分钟单个令牌每分钟最大请求数
每令牌 TPM100,000 tokens/分钟单个令牌每分钟最大 Token 消耗
并发连接数10单个令牌同时进行的请求数
💡 提示 如需更高的速率限制，请联系管理员调整。

# 处理限流响应

触发限流时返回 429，建议实现指数退避重试：

复制 python 
import time, random

def call_with_retry(client, max_retries=5, **kwargs):
    for attempt in range(max_retries):
        try:
            return client.chat.completions.create(**kwargs)
        except Exception as e:
            if getattr(e, 'status_code', None) == 429:
                wait = (2 ** attempt) + random.uniform(0, 1)
                print(f"限流，{wait:.1f}s 后重试...")
                time.sleep(wait)
            else:
                raise
    raise Exception("达到最大重试次数")

# 常见问题

# 和 OpenAI 官方的区别是什么？

因果API 完全兼容 OpenAI API 格式，只需改 base_url。核心优势：一个 Key 调用多个模型供应商、更低的价格、中文客服支持。

# 支持哪些支付方式？

支持支付宝、微信支付在线充值。如有大额或企业对公需求请联系客服。

# 如何查看我的使用量？

登录控制台 → 日志页面，可查看每次请求的模型、Token 消耗、费用等详细信息。

# API Key 泄露了怎么办？

立即登录控制台 → 令牌管理，删除泄露的令牌并创建新的。建议定期轮换。

# 数据传输是否加密？

所有 API 通信均通过 HTTPS 加密传输。
