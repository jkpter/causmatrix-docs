因果API 开发文档

因果API 是新一代 AI 模型中转服务平台，将多种大语言模型统一为 OpenAI 兼容格式。一个 API Key，调用全球主流 AI 模型。

# 项目介绍

因果API 致力于提供稳定、快速、经济的 AI 模型 API 中转服务，让开发者只需接入一次，即可使用 OpenAI、Claude、Gemini、DeepSeek 等数十种主流大模型。

🔌

与 OpenAI Chat Completions API 格式 100% 兼容，零改动迁移。

🌐

多模型汇聚

一个 API Key 访问 OpenAI、Claude、Gemini、DeepSeek 等数十种模型。

⚡

低延迟加速

💰

按量计费、无隐藏费用，实时查看每次调用的 Token 消耗与费用明细。

# 快速开始

只需三步，即可通过因果API 调用全球 AI 大模型。

  1. 注册账号
    访问 causmatrix.xyz 注册并登录你的因果API 账号。
  2. 获取 API Key
    进入控制台 → 令牌管理，创建一个新的 API Key。请立即复制保存——Key 仅在创建时显示一次。
  3. 发起调用
    将 base_url 设为 https://api.causmatrix.xyz/v1，搭配你的 API Key 即可调用。
📋 连接信息
OpenAI API 地址https://api.causmatrix.xyz/v1
Gemini API 地址https://gemini.causmatrix.xyz
接口格式OpenAI 兼容（Chat Completions / Images）
鉴权方式Authorization: Bearer <API-Key>

# cURL

复制 bash 
curl https://api.causmatrix.xyz/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4o",
    "messages": [
      {"role": "system", "content": "你是一个有帮助的助手。"},
      {"role": "user", "content": "你好，请介绍一下自己。"}
    ]
  }'

# Python

复制 python 
from openai import OpenAI

client = OpenAI(
    api_key="YOUR_API_KEY",
    base_url="https://api.causmatrix.xyz/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一个有帮助的助手。"},
        {"role": "user", "content": "你好，请介绍一下自己。"}
    ]
)

print(response.choices[0].message.content)

# 获取 API Key

API Key 是调用因果API 服务的唯一凭证，请妥善保管。

# 创建令牌

  1. 登录因果API，进入 控制台 → 令牌管理
  2. 点击 新建令牌
  3. 填写令牌名称、过期时间、额度限制、模型权限等信息
  4. 点击创建，立即复制 Key 并安全保存
⚠️ 重要：API Key 仅在创建时完整显示一次，关闭弹窗后无法再次查看。如果遗失，只能删除并重新创建。

# 令牌配置项

配置项说明
令牌名称用于标识用途，如「生产环境」「个人测试」
过期时间留空或设为 -1 表示永不过期
剩余额度限制该令牌可消耗的最大金额，超出后自动失效
无限额度开启后不受该令牌额度限制（仅受账户总余额约束）
模型限制限定该令牌可调用的模型，留空不限制
IP 白名单限制可访问的来源 IP，多个用逗号分隔

# 额度与计费

因果API 采用按量计费模式，不同模型有不同的计费倍率。

# 计费规则

  • 账户充值后获得余额（人民币），1 元对应一定额度
  • 不同模型有不同倍率，具体以控制台实时显示为准
  • 费用 =（输入 Token × 输入价格 + 输出 Token × 输出价格）× 倍率
  • 在控制台 → 日志中可查看每次请求的详细扣费记录

# Token 估算参考

  • 英文：1 Token ≈ 0.75 个单词 / 约 4 个字符
  • 中文：1 Token ≈ 1 个汉字
  • 代码：1 Token ≈ 3~4 个字符
  • 一篇 1000 字的中文文章 ≈ 1000 tokens

# 最佳实践

遵循以下建议，获得更安全、高效的使用体验。

# 安全建议

  • 不要在前端代码中直接暴露 API Key——始终通过后端代理转发
  • 为不同环境创建独立的令牌（开发 / 测试 / 生产）
  • 启用 IP 白名单，限制可访问的来源 IP
  • 定期轮换 API Key，降低泄露风险
  • 设置合理的额度上限，防止意外超支

# 性能优化

  • 使用流式输出（stream: true）减少首字等待时间
  • 合理设置 max_tokens，避免不必要的长输出
  • 复用 HTTP 连接（Keep-Alive），减少握手开销
  • 请求失败时实现指数退避重试策略

# 鉴权方式

所有 API 请求均需在 Header 中携带 API Key 进行身份验证。

# Bearer Token（推荐）

在 HTTP Header 中添加 Authorization 字段，与 OpenAI 官方完全一致：

http 
Authorization: Bearer YOUR_API_KEY
⚠️ 注意：鉴权失败时返回 401 Unauthorized。请检查 Key 是否正确、是否已过期、额度是否已耗尽。

# 聊天补全

核心对话接口，兼容 OpenAI Chat Completions API 格式。

POST /v1/chat/completions

向 AI 模型发送对话消息并获取回复。

# 请求参数

参数类型必填说明
modelstring✅模型 ID，如 gpt-4o
messagesarray✅对话消息列表，每条含 role 和 content
temperaturenumber—采样温度 0~2，默认 1。越高越随机
top_pnumber—核采样 0~1，默认 1
max_tokensinteger—最大输出 Token 数
streamboolean—是否流式输出，默认 false
stopstring/array—停止词，遇到即停止生成
toolsarray—函数调用工具定义
tool_choicestring/object—工具调用策略
response_formatobject—输出格式，如 JSON 模式

# 消息角色

角色说明
system系统提示词，设定助手行为与角色
user用户消息
assistant助手回复（多轮对话中回传）
tool工具调用结果

# 流式输出

设置 "stream": true 启用 SSE（Server-Sent Events）流式输出：

复制 python 
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "讲个故事"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")

# 错误码

API 使用标准 HTTP 状态码，并在响应体中返回详细错误说明。

状态码错误类型说明
400invalid_request_error请求参数有误，请检查格式
401authentication_errorAPI Key 无效或缺失
402insufficient_quota余额不足或令牌额度已耗尽
403permission_error无权访问该模型或资源
404not_found_error请求的资源不存在
429rate_limit_error请求频率超限，请稍后重试
500server_error服务器内部错误，可稍后重试或联系支持
503service_unavailable服务暂时不可用

# 处理限流响应

触发限流时返回 429，建议实现指数退避重试：

复制 python 
import time, random

def call_with_retry(client, max_retries=5, **kwargs):
    for attempt in range(max_retries):
        try:
            return client.chat.completions.create(**kwargs)
        except Exception as e:
            if getattr(e, 'status_code', None) == 429:
                wait = (2 ** attempt) + random.uniform(0, 1)
                print(f"限流，{wait:.1f}s 后重试...")
                time.sleep(wait)
            else:
                raise
    raise Exception("达到最大重试次数")

# 常见问题

# 和 OpenAI 官方的区别是什么？

因果API 完全兼容 OpenAI API 格式，只需改 base_url。核心优势：一个 Key 调用多个模型供应商、更低的价格、中文客服支持。

# 支持哪些支付方式？

支持支付宝、微信支付在线充值。如有大额或企业对公需求请联系客服。

# 如何查看我的使用量？

登录控制台 → 日志页面，可查看每次请求的模型、Token 消耗、费用等详细信息。

# API Key 泄露了怎么办？

立即登录控制台 → 令牌管理，删除泄露的令牌并创建新的。建议定期轮换。

# 支持 Claude / Gemini 的原生格式吗？

当前统一使用 OpenAI 兼容格式。使用 OpenAI SDK 将 base_url 指向因果API 即可调用所有支持模型。

# 数据传输是否加密？

所有 API 通信均通过 HTTPS 加密传输。
